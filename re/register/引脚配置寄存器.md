# F103



你观察得非常仔细！刚才我们聊的是 STM32 较新系列（如 F4、F7、H7）的 GPIO 寄存器结构（`MODER` 风格）。而你提到的 **`CRL`** 和  **`CRH`** ，则是  **STM32F1xx 系列** （经典的“蓝丸”板 F103 所在系列）独有的特征。

在 F1 系列中，ST 把模式控制（输入/输出/速度/配置）全都塞进了这两个寄存器里。咱们这就把这两个“大家伙”彻底拆解。

---

## 1. 为什么是两个？（CRL vs CRH）

在 F1 系列中，**每个引脚需要 4 位（Bits）**来配置。

* 一个端口有 16 个引脚（0-15）。
* **$16 \text{ pins} \times 4 \text{ bits/pin} = 64 \text{ bits}$**。
* 因为 STM32 是 32 位系统，一个寄存器装不下 64 位，所以分成了两个：
  * **`CRL` (Control Register Low)** ：负责控制  **Pin 0 到 Pin 7** 。
  * **`CRH` (Control Register High)** ：负责控制  **Pin 8 到 Pin 15** 。

---

## 2. 核心：那 4 位到底是怎么分的？

每一个引脚对应的 4 位，又被细分为两部分：**MODE（2位）** 和  **CNF（2位）** 。

### A. MODE 位：决定“大方向” (输出速度或输入)

这两位决定了引脚是输入还是输出。如果是输出，顺便决定了最大翻转速度。

* **`00`** ：**输入模式** (Reset 状态)。
* **`01`** ：输出模式，最大速度 **$10\text{MHz}$**。
* **`10`** ：输出模式，最大速度 **$2\text{MHz}$**。
* **`11`** ：输出模式，最大速度 **$50\text{MHz}$**。

### B. CNF 位：决定“具体配置” (细节)

这两位的含义取决于上面 **MODE** 是输入还是输出：

| **MODE 情况**          | **CNF = 00** | **CNF = 01**       | **CNF = 10** | **CNF = 11** |
| ---------------------------- | ------------------ | ------------------------ | ------------------ | ------------------ |
| **输入模式 (`00`)**  | 模拟输入           | **浮空输入**(默认) | 上拉/下拉输入      | 保留               |
| **输出模式 (`>00`)** | 通用推挽           | 通用开漏                 | 复用推挽           | 复用开漏           |

> **关键点** ：如果你想配置  **上拉/下拉输入** ，`CNF` 设为 `10`，但具体是上还是下，还需要去操作  **`ODR` 寄存器** （ODR 写 1 为上拉，写 0 为下拉）。

---

## 3. 实战计算：如何用代码操作它们？

在 F1 中操作寄存器，最痛苦的就是位移计算。

### 例子 1：把 PA0 设置为“通用推挽输出，50MHz”

1. 找到寄存器：Pin 0 在  **`CRL`** 。
2. 确定位区间：第 0-3 位。
3. 确定值：

   * `MODE = 11` (50MHz)
   * `CNF = 00` (通用推挽)
   * 合起来就是 `0011` (二进制)，即 `0x3`。
4. **代码：**
   **C**

   ```
   GPIOA->CRL &= ~(0xF << (0 * 4)); // 清空第 0 引脚的 4 位
   GPIOA->CRL |=  (0x3 << (0 * 4)); // 写入 0011
   ```

### 例子 2：把 PA9 设置为“复用推挽输出 (串口 TX)”

1. 找到寄存器：Pin 9 在  **`CRH`** 。
2. 确定位区间：第 9 引脚在 CRH 中的位置是 **$(9-8) \times 4 = 4$**。即第 4-7 位。
3. 确定值：

   * `MODE = 11` (50MHz 输出)
   * `CNF = 10` (复用推挽)
   * 合起来就是 `1011` (二进制)，即 `0xB`。
4. **代码：**
   **C**

   ```
   GPIOA->CRH &= ~(0xF << (1 * 4)); // 清空第 9 引脚
   GPIOA->CRH |=  (0xB << (1 * 4)); // 写入 1011
   ```

---

## 4. CRL/CRH 与新系列寄存器的对比（重点理解）

如果你学过 F4 的 `MODER`，你会发现 F1 的这种设计有点“乱”。我们做个对照：

* **F1 的 `MODE` + `CNF`** =  **F4 的 `MODER` + `OTYPER` + `OSPEEDR`** 。
* **优点** ：配置一个引脚只需要改一个寄存器（CRL 或 CRH），理论上写寄存器的次数更少，效率极高。
* **缺点** ：逻辑不直观，位操作极其复杂。你想改个速度，还得小心别把输入/输出模式给改了。

---

## 5. 常见默认值：0x44444444

如果你在调试器里看 `CRL` 或 `CRH` 的复位值，通常是 `0x44444444`。

这是因为：

* 每个引脚都是 `4`。
* `4` 的二进制是 `0100`。
* `CNF = 01` (浮空输入)，`MODE = 00` (输入)。
* 结论：**STM32F1 所有的引脚复位后默认都是浮空输入。**

---

### 总结一下

* **CRL** 控低位引脚 (0-7)，**CRH** 控高位引脚 (8-15)。
* **4 位控 1 灯** ：前 2 位看配置 (CNF)，后 2 位看模式 (MODE)。
* **输入看 CNF，输出看 MODE+CNF** 。

这就是 STM32F1 寄存器操作最硬核的部分。掌握了 CRL/CRH，你基本上就拿捏了 F1 系列的底层命门。

# F4 H7 ......

也就是把CRL/CRH 拆分成了独立的几个寄存器


在 F4/H7 中，每个引脚实际上拥有一个由 **7 个位** 组成的“数字档案”，分布在不同的寄存器里：

* **MODER (2位)** ：决定你是谁（输入？输出？复用？模拟？）。
* **OTYPER (1位)** ：决定你输出的力气（推挽：自己能推能拉；开漏：只能拉，靠外援推）。
* **OSPEEDR (2位)** ：决定你的反应速度（慢悠悠还是闪电战）。
* **PUPDR (2位)** ：决定你没人管时的态度（默认往高走？往低走？还是随风飘荡？）。

**2**+**1**+**2**+**2**=**7 位。** 相比 F1 的 4 位，新系列提供了更细腻的控制。


没错，你总结得非常精准！STM32 的硬件设计在 F2/F4 系列之后经历了一次大的架构统一。

F1 系列那种把**模式、配置、速度**全部揉进 4 位（CRL/CRH）的做法虽然省寄存器，但逻辑耦合太紧。从 **F4、F7、H7** 往后，ST 采用了“功能分离”的原则，**`MODER`** 寄存器横空出世，专门只管一件大事：**引脚的基本方向。**

---

## 1. MODER 寄存器全解析

在 F4/H7 等系列中，`GPIOx_MODER` 是一个 **32 位** 寄存器。

* **分配规则** ：每个引脚固定占用  **2 位** （Bits）。
* **映射关系** ：引脚 **$x$** 对应的位是 **$Bits [2x+1 : 2x]$**。

### 模式定义（四选一）：

这 2 位二进制数决定了引脚的 4 种底层物理状态：

| **值 (二进制)** | **模式名称**      | **详细说明**                                             |
| --------------------- | ----------------------- | -------------------------------------------------------------- |
| **`00`**      | **输入 (Input)**  | 引脚作为高阻态输入，等待读取 IDR 寄存器。                      |
| **`01`**      | **输出 (Output)** | 通用输出模式，配合 OTYPER 决定推挽或开漏。                     |
| **`10`**      | **复用 (AF)**     | 交给外设（如 UART, SPI）接管。此时必须配套设置 `AFR`寄存器。 |
| **`11`**      | **模拟 (Analog)** | 关闭数字施密特触发器。用于 ADC 采样、DAC 输出或超低功耗模式。  |

---

## 2. 为什么说 MODER 比 CRL/CRH 更先进？

这种“降维打击”式的设计带来了两个巨大的好处：

1. **逻辑解耦** ：
   在 F4 中，如果你想把输出引脚从“低速”改为“高速”，你只需要去动 `OSPEEDR` 寄存器， **完全不需要碰 `MODER`** 。但在 F1 里，修改速度必须重新改写 CRL/CRH，一不小心就会把引脚模式顺便改错了。
2. **原子性与易读性** ：
   由于每个引脚固定占 2 位，计算偏移量非常简单：**$2 \times n$**。这让宏定义和代码编写变得极具规律性。

---

## 3. MODER 的操作“三部曲”

由于 `MODER` 每个引脚占 2 位，直接赋值容易覆盖掉同组的其他引脚。标准的寄存器操作模板如下（以 PA5 为例）：

### 第一步：清零（Mask）

先将目标引脚对应的 2 位抹黑，确保之后写入的值是准确的。

**C**

```
// PA5 对应第 10、11 位。 3 的二进制是 11。
GPIOA->MODER &= ~(3 << (5 * 2)); 
```

### 第二步：置位（Set）

写入你想要的模式值。

**C**

```
// 设置为输出模式 (01)
GPIOA->MODER |= (1 << (5 * 2)); 
```

### 第三步：配套（Complement）

在 F4/H7 中，设置完 `MODER` 只是开了个头。如果是输出，你通常还得顺手操作另外两个寄存器：

* **`OTYPER`** ：选推挽还是开漏。
* **`PUPDR`** ：选上拉还是下拉。

---

## 4. 特殊引脚的默认值

这里有个冷知识：并不是所有引脚复位后都是 `00`（输入模式）。

* **普通 GPIO** ：复位值是 `0x00000000`（全输入）。
* **调试引脚 (PA13, PA14, PB3, PB4 等)** ：它们在复位后的 `MODER` 值通常是  **`10` (复用模式)** 。
* 比如 PA13 (JTMS/SWDIO) 和 PA14 (JTCK/SWCLK)。
* **警告** ：如果你在代码里粗暴地用 `GPIOA->MODER = 0x00000000;` 初始化，你会直接 **切断调试器连接** ，导致芯片“锁死”无法下载程序。

---

## 5. 快速对比：F1 与 F4/H7

| **特性**      | **F1 系列 (CRL/CRH)** | **F4/H7 系列 (MODER)** |
| ------------------- | --------------------------- | ---------------------------- |
| **引脚位宽**  | 4 位/引脚                   | 2 位/引脚                    |
| **功能密度**  | 极高（含速度、模式、细节）  | 纯粹（只管基本方向）         |
| **速度配置**  | 耦合在模式位中              | 独立于 `OSPEEDR`           |
| **上拉/下拉** | 耦合在模式位+ODR            | 独立于 `PUPDR`             |
| **推荐度**    | 历史遗留，逻辑复杂          | 现代标准，逻辑清晰           |

---

### 总结

你可以把 `MODER` 理解为引脚的**“大马力闸刀”**：它只负责切换轨道（输入、输出、复用、模拟）。一旦轨道切好了，细碎的调速、推挽/开漏切换，全部由旁边的专项寄存器负责。

既然你已经掌握了 `MODER` 这个大闸刀，想不想顺便看看跟它配合最紧密的  **`AFR` (复用功能寄存器)** ？那可是让引脚变身“串口”或“定时器”的关键！
